The Multilevel Backup Tool for ZFS

Consitent Snapshots

ZnapZend uses the built-in snapshot functionality of ZFS for fully consistent backups. Many other filebased backup systems will not succeed in capturing a coherent state of a fast changing set of data.

Fileset-Based Configuration

ZnapZend stores all its congfiguration in custom zfs properties. So all configuration is hooked right into the zfs fileset structure. When you modify the fileset, rename it, remove it, znapzend will happily jug along. ZnapZend comes with a special tool for setting and checking the backup configuration.

Remote Backup Locations

ZnapZend uses the ZFS send/receive functionality to transfer backups to remote locations. It employs ssh and (optionally) mbuffer for the transfer to achieve optimal transfer rates for the data. You can configure any number of remote destinations for a fileset. Even no destinations at all if all you need are well managed local snapshots.

Progressive Thinning

ZnapZend can be configured to remove snapshots as they get older. The configuration allows to easily setup schemes where the backup density is progressively reduced as the backups get older. This technique can be individually applied both to the local snapshots as well as to any remote locations.

Online Backup Scheduler

ZnapZend operates as a daemon. It organizes all snapshotting and remote data transfer activities. Even when a snapshot takes a long time to transfer to its remote destination, snapshots on the source fileset will continue to be created according to schedule. Multiple snapshots will be transfered as a group to catch up slow destinations.

Flexible Snapshot Naming

ZnapZend is fully configurable in how it names its snapshots. Switching from your current ZFS snaphotting tool to ZnapZend does not cause you to loose your backup history or having to rename all existing snapshots.
